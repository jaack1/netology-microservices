# Задача 2: Логи

Fluent Bit (или Filebeat) — агент сбора логов на хостах

Logstash (опционально) — обработка и маршрутизация логов

Elasticsearch или OpenSearch — центральное хранилище логов

Kibana или OpenSearch Dashboards — интерфейс для поиска и анализа

ELK-стек — де-факто стандарт для логирования в микросервисах. Если нет ограничений по ресурсам, лучше использовать именно его.

# Задача 3: Мониторинг

Для сбора, хранения и анализа метрик можно использовать стек:

Prometheus — сбор и хранение метрик

Grafana — визуализация и аналитика

Node Exporter — метрики хостов

cAdvisor — метрики контейнеров

(опционально) Alertmanager — система оповещений

Этот стек является де-факто стандартом для мониторинга микросервисных систем и широко применяется в облачных и контейнерных средах. Можно, конечно, использовать и старый, добрый Zabbix (Glaber), но я обратил внимание на то, что он стал редко встречаться в новых проектах, возможно, что он "выходит из моды" или не отвечает всем требованиям современного мониторинга.
